# Cold-Start Measurement Ledger

Use this when a real case teaches the skill.

## Problem

After an expert guide or user correction arrives, it is easy to forget what the method discovered alone and what the expert supplied. Without that distinction, the skill cannot honestly improve.

## Rule

For each real case, maintain a ledger:

| Finding | Tier (`method-alone|expert-or-guide|live-check|delta-action`) | Evidence | Method change |
|---|---|---|---|
|  |  |  |  |

`method-alone` confirms current skill strength. `expert-or-guide` exposes gaps. `delta-action` becomes backlog or a promoted rule.

## Test

Every promoted lesson links to a ledger row. If the skill looked smarter only after a guide, mark it honestly.
